# greedy-lgn — プロジェクトコンテキスト

## これは何か
論理ゲートネットワーク(DLGN)を逆伝播なしで1層ずつ学習する研究プロトタイプ。
各層をローカル損失(GroupSum+CE)で学習→即離散化→凍結→次層はハードな0/1ビット上で学習。
検証精度が頭打ちで層追加を停止(深さ自動決定)。学習後に論理簡略化(定数畳み込み・
パススルー除去・重複マージ・デッドゲート削除)を行い、ビット単位の等価性を検証する。

difflogic(Petersen et al.)系の先行研究に対する位置づけ・関連論文はREADME.mdを参照。

## 検証済みの結果(このリポジトリのexperiment.pyで再現可能)
- sklearn digits, 500ゲート/層, CPU数分:
  - greedy: 深さ4自動選択, ハード回路テスト精度 88.2%, 離散化ギャップ0(構造的)
  - end-to-end逆伝播(同構成): 93.6%(greedyは精度で約5pt負ける — 隠さないこと)
  - 簡略化: 2,000→1,316ゲート(65.8%), 出力完全一致を検証済み, 重複マージは0件
- 小規模設定(--gates 200 --epochs 30)ではe2e側に離散化ギャップ+8.2ptが出現、greedyはゼロ
- CUDA実行(RTX 3060, torch 2.12.1+cu126)はCPUと全数値が完全一致、約10倍高速。
  唯一の例外だったアンサンブルsoft vote(τ除算後のfloat合算が同点クラスの
  丸め順序でデバイス間で割れる)は2026-07-09に整数カウント投票へ修正して解消。
  これに伴う訂正(整数投票の確定値、メンバー精度は全構成ビット一致):
  digits 500×4 soft 91.33→91.11、500×8 soft 91.78→92.00(上下両方向あり)、
  W2J2×4(92.2/92.4)とベスト96.44は不変。MNIST 4構成は全て±0.06pt以内
  (500×4 soft 82.92→82.98、W2J2×4 84.73→84.69、2000skip×4 87.34→87.32、
  4000skip×4=看板 90.86→90.90)。READMEの丸め値90.9/96.4/84.7は全て不変。
  issue #8/#9に訂正コメント投稿済み
- 深さストレステスト(2026-07-08, GPU): e2eは深さ8→12の間でチャンス(10%)に崩壊、
  1200エポックでも回復せず(勾配消失)。greedyは40層目でも学習成立
  (train 69.9% / test 56.0%)。ただしgreedyのテスト精度ピークは深さ4のままで
  以降は情報損失により単調劣化 —「深さで生き残れる≠深さを活かせる」。
  詳細ログはissue #1のコメント、要約はREADMEのDepth stress testセクション
- メモリ等価比較(2026-07-08, GPU, 3シード): greedy 2,000ゲート/層
  (学習ロジット32,000個 = e2e 500×4と同量)で平均95.0% vs e2e 91.5%、
  3シード全勝・バラつきも小さい(94.7〜95.3% vs 90.4〜93.6%)。
  コストは推論回路サイズ約2.7倍(簡略化後約5,300ゲート vs e2e生2,000)。
  幅4倍で初めて重複マージが発生(11〜17件)。ログはissue #1のコメント
- skip connections(2026-07-08, --skip-input実装): 深さ劣化がほぼ解消
  (40層目: 56.0%→83.6%)、ピークも88.2%@4→90.4%@8で初めて深さが精度に貢献。
  幅4倍と併用で3シード平均95.7%(95.6/95.6/96.0)= 単発ネットの自己ベスト
  (全体ベストはアンサンブル96.4%、下記参照)。
  反証された仮説: パススルー約2割は skip でも減らず(利得は情報アクセス由来)。
  ログはissue #1のコメント
- DenseNet式(--skip-all、否定的結果): カーブは最平坦(40層目88.4%)だが
  ピークはskip-inputに届かず(89.8%@29 vs 90.4%@8)、幅4倍でも3シード一貫して
  僅かに劣る(95.1% vs 95.7%)。プール肥大による配線希釈が原因と推測。
  副産物: 簡略化が劇的に効く(40層で76%削除)。推奨は--skip-inputのまま
- MNIST第一歩(2026-07-08, GPU, 1シード): digitsの構図が45倍のデータで再現。
  メモリ等価でgreedy2000+skipがe2e500×4に+4.5pt勝ち(84.6% vs 80.1%)、
  素のgreedy500はe2eに負ける(74.3%)。深さ9自動選択、簡略化73%削減。
  絶対値はdifflogic(~97.7%、ゲート数20倍以上)に遠い小予算の結果 — SOTAでなく
  同予算比較が主張。greedy2000実行770秒。ログはissue #1のコメント
- 先読み窓(2026-07-08, --window/--commit/--win-loss実装): W層をsoftのまま
  逆伝播で共同学習し、先頭J層だけ離散化・凍結して窓をスライド
  (W=1で従来greedyとビット一致 — 回帰テスト済み)。最良はW=2/J=2ブロック+
  --win-loss all: 3シード平均90.4%(全シード勝ち、実行時間ほぼ同等)で、
  e2e(深さ4, 3シードhard平均91.5%)との差が3.1pt→1.0ptに。近視由来の
  ギャップの約2/3を1層の先読みで回収。MNISTでも+2.4pt(74.3→76.6, 深さ6)。
  反証された仮説: (1)--win-loss lastは42%に崩壊 — コミット層が常に「中間層」
  として学習されるため。deep supervision(--win-loss all)必須、
  (2)オーバーラップコミット(J<W, receding horizon)はブロック式(W=J)に
  勝てない — 利得は再計画でなく先読み学習そのもの、(3)W>2は伸びない
  (deep supervisionの希釈+コミット時の共適応破壊と推測)、
  (4)skip-inputとは加算されない(同じ失敗モードを別方向から解いている疑い)。
  エポック2倍対照は不変(88.2%)なので計算量では説明不可。
  W>1では「層境界を勾配が越えない」が「凍結境界を越えない」に弱まる。
  デフォルトはW=1のまま。ログはissue #7
- アンサンブル投票(2026-07-08, --ensemble実装): Mシード分の独立greedyを
  横並べして投票(soft vote=GroupSumカウント合算、多数決=argmax投票+
  カウントでタイブレーク)。他の全レバーと加算される(window×skipは加算
  されなかったのと対照的): digits W2J2×4=92.4%(素の500ゲートで初めて
  e2e平均91.5%超え)、2000+skip×4=多数決96.4%(自己ベスト、従来95.7%)。
  MNISTでは投票利得がdigitsより大きく(+8.4pt vs +3.6pt)、500×4+W2J2の
  soft vote 84.7%が従来ベスト(単発2000+skip 84.6%)を半分の学習メモリ
  (16,000 vs 32,000ロジット)・少ない生ゲート数(13,000 vs 18,000)で上回る。
  正直な限界: 推論面積を揃えると幅の直接拡大が明確に勝つ(digits 4×500=
  91.3% vs 幅2000単発95.0%)— アンサンブルは幅の代替でなく「学習メモリを
  増やさずに推論面積を精度に変換する」レバー。投票規則はメンバーが弱いと
  soft vote優位、強いと多数決優位に逆転。ログはissue #8
- MNISTスケーリング(2026-07-08, 第1・第2スイープ): 幅が支配的レバー
  (4,000ゲート単発89.8%、2,000比+5.2pt、幅未飽和)。アンサンブルは全幅で
  加算されるが利得は逓減(500で+8.4pt→2000で+3.2pt→4000で+1.1pt)。
  ベストは4,000×4のsoft vote 90.86% = 初の90%超え(difflogicとの差13pt→7pt)。
  死にレバー2つを確認: エポック2倍+0.1pt、window×幅×skip+0.1pt。
  hard_batchedのチャンクを幅に反比例する予算制に変更(ビット等価、500ゲートは
  従来と同一チャンク)で4,000ゲートが6GB VRAMに収まる。8,000ゲートは
  OOMで不成立(2026-07-09再実行: 層3作成時にクラッシュ。原因は評価一時
  テンソルでなく常駐プール[60000,10352]float32+遷移時の一時コピー≈7GB)。
  ただし部分結果が有望: 深さ2で90.01% = 4,000ゲート単発ベスト89.77%超え。
  修正案: uint8プール / CPU常駐プール+バッチ転送 / in-placeプールバッファ。
  ログと診断はissue #9のコメント
- Forward-Forward目的(2026-07-09, --objective ff / --ff-label-rep実装):
  goodness=popcount(バイナリ層でHintonの二乗和が退化)、ラベルはone-hotを
  入力に連結、推論は10ラベル試行。最初の失敗: ランダム2入力配線では素の
  10ビット重畳だと約9割のゲートがラベルを見ず18%に沈む → ラベルビット複製
  (--ff-label-rep)で解決、最適はラベル:データ≈2:1(digitsでrep38)。
  digits 3シード平均86.0%(groupsum 88.4%に-2.4pt)、skip-inputはFFには
  有害(83.6→80.2)。MNIST(rep470)は76.8%@深さ17でgroupsum 500の74.3%を
  +2.5pt逆転、しかも17層まで単調増加 = skipなしで深さを活かせた初のレバー
  (groupsumは深さ4〜6でピークアウトしていた)。層内は勾配学習のまま
  (FFは目的関数の置換であり勾配フリー化ではない)。
  FF×先読み窓(W2J2, deep supervision固定): 積まれる — digits 3シード平均
  86.0→88.0%(+2.0pt全シード勝ち、素のgroupsum 88.4%に肉薄)、MNIST
  76.8→78.2%@深さ19(500ゲート単発の最高値)。window×skipが積まれなかった
  のと対照的で、FFの深さ耐性とlookaheadは別の欠損を直している示唆。
  W=1はFFとビット一致(回帰済み)。
  負例マイニング(--ff-neg, ユーザー発案「人間は間違えた所を重点復習する」):
  hard純粋は崩壊(digits 32〜65%、負例分布の偏り)、mix(最難+ランダム半々)は
  digitsで加算 — W1 86.0→88.1%、窓上で88.0→89.7%(素のgroupsum 88.4%超え)。
  ただし「凍結プレフィックスから採点」版はMNISTで不発(77.9 vs 78.2)。
  warmup+review(--ff-neg-warmup, ユーザー発案「まず普通に学習して正答率が
  上がってから誤答を難問として復習」): ユーザーの2洞察が両方的中 —
  (a)warmupでhard崩壊を救済(digits 32→88.0%、未熟な採点者が崩壊原因だった)、
  (b)MNIST不発はマイニング「質」の問題で、窓自身に模試を受けさせるreview+
  warmup0.5がMNIST 82.0%@深さ15(random78.2から+3.8pt=リポジトリの500ゲート
  単発MNIST最高、全目的関数通じて)。digitsは天井で差つかず(mix89.7が僅差最良)、
  MNISTで大差 = digits天井/MNIST審判の2層運用の好例。ログはissue #10のコメント
- 恒等warm-start(--warm-start, 2026-07-13): 各層を「前層出力の再現(gate Aパス
  スルー+末尾恒等配線)」で初期化してから残差を微調整(ResNetの恒等ブロック)。
  digits 3シード: 素88.4→窓90.4に対しwarm5=94.5%(窓を+4.1pt圧倒、深さ15まで
  生産的)。MNISTも窓を+11pt上回るが単発ノイズ±4pt(深さ上限で急勾配のため)、
  残差champ(90.9%)には未達(0.87@深さ20)。「素greedyを深さ生産的にし先読み窓を
  葬るレバー、ただし残差の代替ではない」。residual+warm5は残差単体と横ばい
  (boostと同じ、治す病が重複)。RESULTS.md「Identity warm-start」節
- 残差×誤答重点(--group-boost, 2026-07-13): 残差の上でAdaBoost式サンプル再重み
  付け。MNIST 3シード平均+0.43pt(B=3、2/3シード勝ち)の控えめレバー。残差が
  既に暗黙に誤答重視のため上乗せ小。看板の残差+skip(93.85%)は不変
- 適応エポック(--epoch-stop/peak/decay/chain, 2026-07-13): 固定120でなくargmax
  変化率(離散回路のchurn)で層を早期終了。否定的結果 — digits+warmで全変種が
  固定120(95.6%)に未達。churn半減点が~125epochで固定120が既にほぼ最適。
  --epoch-chain 2は固定120に並びつつ分散5分の1(安定性の勝ち)。知見: 層を完全
  飽和させると深さ成長が止まる(飽和層の微小利得が深さpatienceを踏む)=「1層の
  収束はスタックの目的でない」。RESULTS.md「Adaptive per-layer epochs」節
- 再帰2種(--recur / --seq, 2026-07-13): 重み共有再帰。--recur K=静的入力に層を
  K回反復(パラメータ圧縮、simplifyは展開回路で等価検証)、--seq=RDDLGN型時系列
  (画像を行ごと提示 s_t=L([x_t;s_{t-1}])、BPTT、GroupSum読出、配線はarXiv
  2508.06097と同型・新規性主張なし)。両方とも素の初期化で崩壊し恒等warmで救済。
  --seq digits 3シード: warm3=0.912(毎ステップ24ビットで静的素0.884を+2.8pt、
  静的warm0.945には未達)。鋭い機構: --seqの恒等は「前層状態」向き必須で「自状態」
  向きは崩壊(82→27%、ゼロ初期状態の自己ループ)。3レバー通算の結論=離散論理の
  再帰は恒等に近い写像が前提かつ恒等の向きが本質(RDDLGN Table 7のResidual init
  必須・Gaussian崩壊22.6%と独立に一致)。MNIST --seqは保留(28ステップBPTTが
  6GB GPUで対話的には重すぎ、数時間バッチ要)。RESULTS.md「Within-layer recursion」節
- 入力二値化(--thresholds, 2026-07-13): 診断と逆の処方箋が勝った。分位点整合
  (5,10,15)は負け、「既定に低側の面を足す」(digits 1,3,7,11,15 / MNIST
  31,63,127,191)が勝ち — MNIST残差3シード89.96→90.71(+0.75pt全勝・分散縮小)。
  公開値90.86は別セッション単発なので「同一プロトコル比較で+0.75」とだけ主張。
  RESULTS.md「Input binarization」節
- champ+低側面 = 新記録94.08%(2026-07-13, MNIST seed1): residual+skip+
  thresholds 31,63,127,191 が対照93.82@34に対し**94.08@27** — 精度だけでなく
  浅く(34→27)・小さく(簡略化後13,711→11,005ゲート)・速く(46→29分)。
  simplifyの残差対応によりこの看板は**初のビット等価検証済み**(旧93.85は
  未検証のまま公開していた)。留保: champ比較は1シードで+0.26はノイズ圏の
  可能性(信頼の根拠は残差単体の3シード全勝+0.75の方)。面積/深さ/時間の
  優位は精度ノイズと無関係に成立

## ファイル
2026-07-12にモジュール分割(純粋なコード移動、CLI・全数値ビット等価を回帰で確認):
- experiment.py : CLI入口(argparse+main のみ)。既存コマンドは全て不変
- core.py      : 共有基盤(16ゲート、LogicLayer、load_data、fit、適応エポック停止判定)
- groupsum.py  : GroupSum目的(主線)。residual/boost/warm-startはこの中のモード
- ff.py        : Forward-Forward目的(負例マイニング・構造化配線含む)
- greedy.py    : 手法本体(run_greedy + make_objective)
- scaling.py   : オフトラックのスケーリングレバー(アンサンブル投票、SCALING.md対応)
- e2e.py       : 逆伝播ベースライン
- simplify.py  : 論理簡略化+ビット等価検証
- seq.py       : row-sequential再帰(--seq、RDDLGN型の時系列状態。2026-07-13追加)
- README.md    : 英語本体+日本語概要。要点サマリ+RESULTS.mdへのリンク集に圧縮済み
- RESULTS.md   : 実験詳細の本体(セットアップ・数値表・反証された仮説)。
                 今後の実験結果はREADMEでなくここに追記する
- requirements.txt, LICENSE(MIT, 著作権者名は要確認)

  (GPU約10秒/本、スモーク・3シードの方向性確認・回帰テスト用)、
  MNIST=審判(digitsで有望だった候補だけ確定、約40-50分/本)。
  READMEやRESULTSに載せる主張はMNISTの数字で語る。digitsとMNISTの結論が
  食い違ったらMNISTを信じる(FFの勝敗・マイニングの効き方で実際に食い違った)。

  進める(スケーリングレバー=幅・アンサンブルは「効くと分かっている資源投入」
  なので、アイデアの評価はこの固定予算の土俵で行う)。比較の物差しはREADMEの
  fixed-budget表(MNIST: 素74.3→窓76.6→FF76.8→FF窓78.2→+review復習82.0)。
  例外: 大きな当たりが出たときだけ単発のスケール確認を1本(500での勝敗が
  スケール後も同じとは限らない — window×幅の死、skip×幅の相乗を既に観測)。
